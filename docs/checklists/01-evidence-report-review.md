# 运行过程证据报告检查清单

适用文件：`01_运行过程证据报告.md/html/pdf`

## 内容检查

- [ ] 执行信息完整：系统、地址、租户、账号、日期、压测档位。
- [ ] 页面与功能证据索引覆盖所有主要页面。
- [ ] 每个页面证据都有截图文件或结构化日志。
- [ ] 查询与数据检查证据包含日志、用户、组织、角色、模型/素材、AI 创作、大数据专项。
- [ ] 正式压测配置、结果、阈值、结论齐全。
- [ ] 失败响应样本存在并可反查。
- [ ] 反查文件列表只写相对路径或文件名，不写本机绝对路径。

## 压测专项检查

- [ ] `formal_load_summary.json` 存在。
- [ ] `formal_load_progress.json` 存在。
- [ ] `formal_load_auth_check.json` 存在。
- [ ] 若存在失败请求，`formal_load_failure_responses.json` 必须存在。
- [ ] 失败响应表包含场景、方法、接口、HTTP 状态、失败类型、响应/异常摘要。
- [ ] 关键失败分布图存在并能打开。

## 安全和可读性检查

- [ ] 不暴露密码、手机号、租户码等敏感信息。
- [ ] 不包含 `file://`、`C:\`、`C:/`、用户目录或临时目录。
- [ ] PDF 渲染无空白页、表格截断、乱码、页眉页脚路径。

## 通过标准

证据报告必须能支撑实际交付报告中的每一个结论；任何不能反查的结论都要补证据或降级描述。
