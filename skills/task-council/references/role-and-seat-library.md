# 角色与席位库

## 咨询角色

| 角色 | 倡议席问题 | 审慎席问题 | 落地席问题 |
|---|---|---|---|
| 任务发起/业务 | 目标为何值得投入？ | 是否在解决伪问题或扩大范围？ | 谁负责结果，何时复盘？ |
| 用户与体验 | 用户得到的核心价值是什么？ | 哪个环节最难理解、最易流失？ | 如何验证真实用户行为？ |
| 专业领域 | 哪条专业路径最可行？ | 哪个前提不成立会推翻方案？ | 需要哪些专业输入、环境或评审？ |
| 执行与运营 | 如何以最少步骤实现目标？ | 依赖、容量、维护和交接会怎样失败？ | 里程碑、资源、验收和回滚是什么？ |
| 财务与资源 | 哪种投入组合收益最好？ | 隐性成本、机会成本和现金风险是什么？ | 预算、采购、资源审批如何安排？ |
| 风险与合规 | 如何在规则内实现目标？ | 哪些安全、隐私、合同或法规问题可阻断？ | 审批、留痕、应急与审计如何执行？ |
| 市场与外部情报 | 外部变化提供什么机会？ | 证据是否过时、不可比或被竞品误导？ | 还需调查哪些一手来源和时间点？ |
| 反方/红队 | 更简单的替代方案是什么？ | 方案会怎样失败、被滥用或造成二阶伤害？ | 什么指标或事件应立即停止？ |

## 席位独立性规则

- 不得让同一角色的两个席位共享同一份结论后再改写语气。
- 倡议席不得兼任反方/红队的审慎席；决策负责人不得担任独立反方席。
- 每席首轮只回答自己的问题，写出结论、证据、假设、风险与建议动作。
- 同一角色多个席位应使用不同的证据或推理方法；若只能重复，应减少席位数量。

## 透镜分配

将人物方法论分配给席位，而不是创建脱离任务的名人角色：

| 透镜 | 优先分配 | 作用 |
|---|---|---|
| Jobs | 用户与体验倡议席、任务发起倡议席 | 聚焦核心价值，移除复杂性 |
| Munger | 财务与资源审慎席、反方席 | 检查激励、二阶效应与逆向失败 |
| Taleb | 风险与合规审慎席、反方席 | 识别尾部风险、脆弱性和不可逆损失 |
| Zhang Yiming | 用户与体验落地席、市场与外部情报落地席 | 设计小实验、数据反馈与迭代 |
| Musk | 专业领域落地席、执行与运营落地席 | 从真实约束、成本和能力反推方案 |
