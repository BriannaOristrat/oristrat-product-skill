# 外部视角与 Council 来源边界

## 使用原则

- 先执行本 Skill 的会议编制、证据和异议规则；外部 Skill 只补充某一个席位的方法，不替代会议流程。
- 只在当前环境实际可发现时调用外部或系统 Skill。缺失时，使用本文件中的透镜问题继续，不将会议标记为阻塞。
- 不复制外部 Skill 正文，不把 `external-skills/` 缓存作为正式运行入口。
- 人物视角是角色提示词和检查问题，不是事实来源，也不意味着历史人物真实参与或认可结论。

## 可选视角

| 视角 | 优先用途 | 不可替代的证据 |
|---|---|---|
| `steve-jobs-perspective` | 简化产品、任务和沟通 | 用户研究、可用性证据 |
| `munger-perspective` | 激励、反向推演、二阶影响 | 成本、行为、制度或历史数据 |
| `taleb-perspective` | 尾部风险、脆弱性、止损 | 风险评估、容量或事故证据 |
| `zhang-yiming-perspective` | 实验设计、指标与迭代 | 实验数据、指标定义 |
| `elon-musk-perspective` | 第一性约束、成本和可行性 | 技术、物理、资源或财务事实 |

## 外部 Council 方法参考

仓库的 `external-skills/ercoledevs-codex-council/` 可作为来源追溯，借鉴独立首轮、匿名候选评审、保留异议、预估工作量和验证优先等方法。它不是 `task-council` 的执行依赖；若未来正式安装该插件，可在用户确认后将其作为额外的高风险评审层。
