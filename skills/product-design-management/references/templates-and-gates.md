## 用户故事模板

```markdown
As a <用户角色>,
I want to <完成的任务>,
so that <获得的价值>.

Acceptance Criteria:
- Given <前置条件>, when <用户行为>, then <系统结果>.
- Given <异常条件>, when <用户行为>, then <系统反馈>.
```

## Roadmap 决策表

| 候选项 | 用户价值 | 商业价值 | 证据强度 | 复杂度 | 风险 | 推荐 |
|---|---:|---:|---:|---:|---:|---|
|  |  |  |  |  |  |  |

评分后必须写出：

- 推荐做什么。
- 暂缓什么。
- 最大不确定性是什么。
- 下一步如何降低不确定性。

## Handoff 门禁

- 有明确 owner 和目标发布时间。
- 主流程和异常流程完整。
- 每个关键页面/状态有说明。
- 新增组件、字段、权限、埋点已列出。
- 验收标准可由测试或研发验证。
- 风险、依赖和未决问题单独列出。

## 最终输出门禁

完成前逐项检查：

- 是否能让团队理解为什么做。
- 是否能让设计理解用户和流程。
- 是否能让研发知道要实现什么。
- 是否能让测试知道如何验收。
- 是否能让业务方知道取舍和预期结果。
- 是否没有泄露敏感信息、账号、token、本地路径或客户隐私。
