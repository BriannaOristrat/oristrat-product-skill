# 04 交付包生成与归档工作流

## 目标

把最终报告、必要证据和附件打包成可交付、可审计、无临时污染的资料包。

## 归档根目录

默认归档根目录：`D:\AAAA\资料归档\codex_files\00_入口\01_产物\输出\报告`。

如果该本地归档根不可用，回退到 `自研skill\输出\报告`。最终报告文件夹和 zip 必须位于归档根目录下；不得默认交付到当前 C 盘工作区。

## 标准目录

```text
<归档根目录>/<单次交付目录>/
  01_运行过程证据报告.md
  01_运行过程证据报告.html
  01_运行过程证据报告.pdf
  02_实际交付报告.md
  02_实际交付报告.html
  02_实际交付报告.pdf
  03_五轮业务论证与结果.md
  03_五轮业务论证与结果.html
  03_五轮业务论证与结果.pdf
  appendix/
  evidence/
```

## 打包规则

- 包含最终 `.md/.html/.pdf`。
- 包含可反查证据和必要附件。
- 不包含错误版、临时版、被占用无法覆盖的旧版。
- 不包含小于 2KB 的异常 PDF。
- 不包含 `.inspect.ndjson`、缓存、临时目录。
- 压测结果只保留正式档和必要失败响应，不塞入全部实验性压测过程。

## 清理规则

交付前检查并删除：

- `*_新版.pdf`
- `*_测试报告.pdf`
- 浏览器打印带页眉页脚的错误 PDF
- 临时渲染 PNG
- 临时脚本输出

如果文件被 WPS 或 PDF 阅读器占用，先关闭对应窗口再清理。

## 归档门禁

执行 [最终交付门禁](../checklists/05-final-delivery-gate.md) 后再交付 zip。
