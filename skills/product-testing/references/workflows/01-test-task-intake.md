# 01 测试任务启动与范围确认

## 目标

在执行任何测试前，把测试对象、环境、账号、数据、风险和允许动作说清楚，避免后续报告无法解释边界。

## 输入

| 输入 | 说明 |
|---|---|
| 测试对象 | 系统、模块、版本、入口 URL |
| 测试目标 | 初验、回归、专项、压测、问题复核 |
| 账号和权限 | 租户、角色、账号、是否允许写入 |
| 环境信息 | 测试/预发/生产、网络、部署窗口 |
| 数据策略 | 是否允许造数、是否需要清理、唯一前缀 |
| 验收口径 | 通过/部分通过/不通过/阻塞定义 |

## 执行步骤

1. 明确测试任务名称和交付对象。
2. 确认测试范围和不覆盖范围。
3. 确认账号、租户、权限边界。
4. 确认可写入动作：创建、修改、删除、压测。
5. 确认证据路径：截图、日志、接口响应、压测结果。
6. 确认最终报告类型：运行过程证据报告、实际交付报告、业务论证报告。

## 输出

- 测试任务范围说明。
- 风险和不覆盖项说明。
- 证据采集清单。
- 允许写入和清理策略。
