# 运行过程证据报告检查清单

适用文件：`01_运行过程证据报告.md/html/pdf`

## 内容检查

- [ ] 执行信息完整：系统、地址、租户、账号、日期、压测档位。
- [ ] 页面与功能证据索引覆盖所有主要页面。
- [ ] 每个页面证据都有截图文件或结构化日志。
- [ ] 查询与数据检查证据包含日志、用户、组织、角色、模型/素材、AI 创作、大数据专项。
- [ ] 正式压测配置、结果、阈值、结论齐全。
- [ ] 失败响应样本存在并可反查。
- [ ] 反查文件列表只写相对路径或文件名，不写本机绝对路径。

## 02 异常数据对应关系

- [ ] `02_实际交付报告` 中每个异常、观察项、部分通过、不通过、阻塞项，都能在 `01_运行过程证据报告` 中找到对应证据。
- [ ] 正式压测异常必须能反查到 `formal_load_summary.json`、`formal_load_progress.json`、`formal_load_failure_responses.json` 和关键失败分布图。
- [ ] 若 `02` 展示错误率、失败样本、P95 / P99、最大响应或分场景失败表，`01` 必须保留同源指标、明细表和源文件路径。
- [ ] 页面可见错误、空白页、接口错误提示必须有错误截图；若因接口/日志类异常无法截图，必须写明缺失原因和替代证据。
- [ ] 证据完整性必须覆盖：截图是否存在、源 JSON 是否存在、失败响应是否可读、报告结论是否可反查。

## 压测专项检查

- [ ] `formal_load_summary.json` 存在。
- [ ] `formal_load_progress.json` 存在。
- [ ] `formal_load_auth_check.json` 存在。
- [ ] 若存在失败请求，`formal_load_failure_responses.json` 必须存在。
- [ ] 失败响应表包含场景、方法、接口、HTTP 状态、失败类型、响应/异常摘要。
- [ ] 关键失败分布图存在并能打开。

## 安全和可读性检查

- [ ] 不暴露密码、手机号、租户码等敏感信息。
- [ ] 不包含 `file://`、`C:\`、`C:/`、用户目录或临时目录。
- [ ] PDF 渲染无空白页、表格截断、乱码、页眉页脚路径。

## 通过标准

证据报告必须能支撑实际交付报告中的每一个结论；任何不能反查的结论都要补证据或降级描述。
