# 实际交付报告检查清单

适用文件：`02_实际交付报告.md/html/pdf`

## 标题和定位

- [ ] 正文主标题必须是 `测试报告`。
- [ ] 文件名可保留 `02_实际交付报告`，但正文不再使用“实际交付报告”作为标题。
- [ ] 面向客户/业务读者，避免过程化、调试化表达。

## 内容检查

- [ ] 文档信息包含系统名称、测试日期、测试环境、测试账号、结论。
- [ ] 测试范围清楚说明已覆盖和边界。
- [ ] 执行结果汇总包含功能验证、正式性能压测、大数据量专项。
- [ ] 主要通过项和部分通过项没有互相矛盾。
- [ ] 阻塞项为空时明确写“无”，不留空表。
- [ ] 性能专项说明阈值、实测结果、是否通过。
- [ ] 大数据量专项说明造数、验证、清理结果。
- [ ] 初验结论一句话能被管理层直接引用。

## 路径和安全检查

- [ ] 正文图片使用相对路径，例如 `evidence/.../file.png`。
- [ ] 不出现 `file://`。
- [ ] 不出现 `C:\` 或 `C:/`。
- [ ] 不出现 `Users/`、`Users\`、用户名、临时目录。
- [ ] PDF 没有浏览器默认页眉页脚。
