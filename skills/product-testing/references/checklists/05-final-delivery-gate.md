# 最终交付门禁

## 目标

确保交付目录中只保留最终可交付版本，并且报告结论、证据和打包内容一致。

## 文件检查

- [ ] 最终报告文件夹和 zip 位于 `ORISTRAT_REPORT_ARCHIVE_ROOT` 指定的本机归档根目录；该变量可来自本机环境变量或被忽略的 `.env.local`。如该归档根不可用，则位于 `自研skill\输出\报告`。
- [ ] 仓库正文和正式报告中没有写入个人机器的绝对路径。
- [ ] 最终交付位置不是当前工作区，除非用户明确指定。
- [ ] `01_运行过程证据报告.md/html/pdf` 存在。
- [ ] `02_实际交付报告.md/html/pdf` 存在。
- [ ] `03_五轮业务论证与结果.md/html/pdf` 存在。
- [ ] `appendix/` 存在且附件可打开。
- [ ] `evidence/` 存在且关键证据可反查。
- [ ] zip 文件存在并能打开。

## 清理检查

- [ ] 无 `*_新版.pdf`。
- [ ] 无 `*_测试报告.pdf` 临时版。
- [ ] 无小于 2KB 的异常 PDF。
- [ ] 无 `.inspect.ndjson`。
- [ ] 无临时渲染图。
- [ ] 无浏览器错误导出的 PDF。

## 一致性检查

- [ ] 三份报告的性能结论一致。
- [ ] 三份报告的大数据专项结论一致。
- [ ] 失败响应样本数量一致。
- [ ] 阻塞项数量一致。
- [ ] 实际交付报告中的每个结论都能在运行过程证据报告里反查。

## 安全检查

- [ ] 无本机绝对路径。
- [ ] 无账号密码明文。
- [ ] 无手机号、租户码等敏感信息明文。
- [ ] 无 Prompt、调试过程、内部临时说明。
- [ ] 若交付内容需要提交 GitHub，已通过仓库级 [GitHub 提交安全门禁](../../../../docs/github-submit-safety-gate.md)。

## 通过标准

如果任何一项未通过，不交付 zip；先修复报告或证据，再重新生成。
